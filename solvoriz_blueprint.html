Section 2

Product Vision

What Solvoriz is, who it serves, and why narrow focus is the strategic advantage.

What Solvoriz Is

Solvoriz is a curated talent discovery platform for students aged 15–21. Students build proof-of-work profiles anchored by real projects. Recruiters search, filter, and discover talent based on what students have actually built — not their GPA, school name, or internship history.

The core unit of value is the Project, not the resume. A student's profile is only as strong as the projects they add. This makes Solvoriz honest, skill-signal-dense, and impossible to fake with buzzwords.

👤

For Students

A simple profile builder where every project is a proof point. Create a public portfolio that speaks louder than a resume.

🔍

For Recruiters

A fast, filter-driven talent browser with a pool recruiters can't access anywhere else — young, active builders, pre-verified.

⚡

Core Engine

Search filters across skill, location, project type, and age. Student cards that show the work first. Fast contact flow.

The Problem It Solves

WhoProblemWhat existsHow Solvoriz fixes it
Students 15–21No professional platform for young, pre-experience talentLinkedIn (too corporate), GitHub (too niche)A profile built for doers, not degree-holders
RecruitersCan't find skilled young talent before they're "hireable"Job boards, campus fairsDirect discovery by skill and project evidence
CompaniesExpensive and slow early-talent pipelinesInternship programs, campus recruitmentOn-demand search with instant shortlisting

What Solvoriz Is NOT

  • Not a job board — there are no job postings in v1
  • Not a LinkedIn clone — no feed, no connections, no social graph
  • Not an ATS — no application tracking or HR workflow
  • Not a portfolio hosting tool — projects are signals, not showcases
  • Not a learning platform — no courses, no certificates, no paths
  • Not a full-age platform — strictly 15–21, the strategic moat

Why Narrow Focus Is Smart

The 15–21 talent pool is massive, underserved, and growing faster than traditional platforms can capture it. By going narrow early, Solvoriz wins a category that LinkedIn has ignored (too young, no work history), GitHub has missed (non-technical creators), and job boards can't serve (no formal employment history yet). The narrow focus is the moat.

User Types

Three roles, three jobs-to-be-done, three distinct success moments.

Role 01

The Student

Goal

Get discovered by companies and internship programs before graduating. Build a verifiable track record of real work.

Main Actions

Create profile → Add projects → Set visibility → Get contacted

Pain Points

No professional platform for their age. Resume doesn't reflect what they can actually do. Zero discoverability.

Success Moment

"A recruiter from [company] messaged me about my project."

Role 02

The Recruiter

Goal

Find skilled young talent fast, before competitors do. Build an early-talent pipeline with less effort.

Main Actions

Set filters → Browse students → View profiles → Shortlist → Contact

Pain Points

Campus hiring is expensive and slow. LinkedIn has no pre-experience talent. No way to evaluate young builders.

Success Moment

"I found 3 students with exactly the right skills in 10 minutes."

Role 03

The Admin/Founder

Goal

Grow a high-quality supply (students) and monetizable demand (recruiters). Maintain trust and curate quality.

Main Actions

Monitor KPIs → Verify students → Feature profiles → Manage moderation → Track revenue

Pain Points

Low-quality profiles kill recruiter trust. Need traction signals fast. Monetization requires supply-demand balance.

Success Moment

"Recruiter paid for premium access because they found a hire on Solvoriz."

Unlock rule: Students only appear in recruiter search results when their profile score reaches 70+. This is the quality gate. Show it clearly on dashboard.

Post-Signup Student Flow

  1. Email verification Verify via link in email. Unverified profiles are not discoverable.
  2. Onboarding splash "Solvoriz helps you get found by real companies through your projects. Let's build your profile." — 2 screen max.
  3. Basic info prompt Name, age (15–21 gate), location, skills. Must complete before proceeding.
  4. First project prompt "Add your best project" modal appears. Cannot skip the first one — it's the product's core value.
  5. Profile preview shown Student sees their public profile. Celebrates with "You're discoverable!" if score ≥ 70.
  6. Dashboard home Profile score bar, projects list, "Add another project" nudge, share profile link button.

How Projects Should Be Added

  • Single-page form: title (required), description (200 char max), skills used (tag select, max 5), project link (URL), image upload (1 image, max 2MB), project type radio (Side Project / School Project / Freelance / Internship)
  • Auto-save draft so students don't lose progress
  • Projects default to visible=true when added
  • Admin can flag a project — student sees a "Project under review" label
  • Max 8 projects in v1 (prevents vanity bloat, keeps quality high)

Student Card Design (Browse Grid)

  • Top 3 skill chips (colored tags)
  • Latest project thumbnail image (or placeholder pattern)
  • Latest project name below thumbnail
  • Shortlist (heart) button top-right of card
  • Featured badge (gold dot) if admin-featured

Contact Flow

  1. Recruiter clicks "Contact [Student]" on full profile page.
  2. Message composer modal Pre-filled with company name. Recruiter writes brief message (280 char max). Uses 1 contact credit.
  3. Student gets email notification "[Company] wants to connect with you on Solvoriz. Click to see their message."
  4. Student responds in dashboard Simple reply thread (2-message max in v1). After that, both sides exchange real emails.
  5. Recruiter sees status in Contact History: Sent → Seen → Replied.

Recruiter Onboarding

  1. Signup with company email Must be corporate domain. Personal Gmail rejected.
  2. Company profile Company name, industry, role title, website.
  3. Pending verification Admin approves within 24h. Email sent on approval.
  4. First browse session Guided tour: filter panel → student card → profile view → contact button.
  5. Free trial 5 contact credits included free. Upgrade to paid plan for more.

Fraud / Spam Detection Ideas (v1: Manual Rules)

  • Flag profiles with identical bio text (copy-paste detection)
  • Flag projects with stock image links or placeholder URLs
  • Flag recruiters signing up with free email providers (Gmail, Yahoo, Outlook)
  • Flag students under 15 or over 21 based on self-reported age
  • Flag profiles with 0 project activity for 14+ days after signup
  • Rate-limit contact requests per recruiter (max 20/day in v1)

Metrics Timeline

First 30 Days — What matters most: Total students discoverable (quality supply), recruiter approval count, contact requests sent (demand signal), profile completion rate, and daily signups. These tell you if the core loop works.
After Product-Market Fit: Recruiter retention rate, contact-to-response rate, time-to-hire signal (survey), revenue per recruiter, referral rate (student and recruiter), NPS.

Section 9

User Flows

Detailed step-by-step flows for every critical action in v1.

Flow 1 — Student Signup to Completed Profile

  1. Land on homepage Clicks "Join as Student" CTA.
  2. Signup Google OAuth or email/password. Age gate question: "How old are you?" — rejects <15 and >21.
  3. Email verification Verification link sent. Must verify to proceed.
  4. Onboarding step 1 Name, handle (auto-suggested), location (city).
  5. Onboarding step 2 Add skills (tag picker, min 3 to proceed).
  6. First project prompt Modal: "Add your first project — this is what companies will see." Required to proceed.
  7. Profile complete Score shown. If ≥70: "You're discoverable!" If <70: checklist of what's missing.
  8. Dashboard home Share profile link nudge. "Add your second project" card.

Flow 2 — Student Adding First Project

  1. Click "Add Project" from dashboard or onboarding modal.
  2. Project form appears Single-page form. Fields: title, description, skills used (tag select), project URL, image upload, project type.
  3. Auto-save draft Every 30 seconds. Prevents data loss.
  4. Submit Project goes live instantly (status: active). No approval needed in v1.
  5. Profile score updates +20 points. Progress bar animates up.
  6. Confirmation "Project added! It's now visible on your profile." with link to public profile.

Flow 3 — Recruiter Signup to First Search

  1. Land on homepage Clicks "Find Student Talent" CTA.
  2. Signup form Name, work email (company domain required), company, role title, website.
  3. Pending screen "We're verifying your account — usually takes under 24 hours."
  4. Admin approves Triggers automated email: "You're in. Start discovering students."
  5. First login Onboarding tooltip: "Use filters left → browse cards right → click card to view full profile."
  6. First filter applied Select a skill. Results appear. Browse begins.
  7. 5 free credits shown Banner: "You have 5 free contact credits. Use them wisely."

Flow 4 — Recruiter Finding and Contacting a Student

  1. Apply filters Skills: "Figma, UI design". Age: 17–19. Location: Delhi.
  2. Browse results 12 student cards appear. Scroll through. Notice a student with a strong project thumbnail.
  3. Click card Full profile opens. Read bio, browse projects, click project link.
  4. Decide to contact Click "Contact Aryan" button.
  5. Message modal Write message (280 char max). Click "Send" — uses 1 credit.
  6. Confirmation "Message sent! Aryan will be notified by email."
  7. Contact history updated Shows "Sent — Pending response."

Flow 5 — Admin Reviewing a Flagged Profile/Project

  1. Flagged item appears in admin moderation queue with reason (auto-flagged or user-reported).
  2. Admin reviews Sees full student profile + projects. Reads flag reason.
  3. Decision Options: Approve (remove flag), Edit (fix minor issue), Suspend (hide from search), Remove (delete with reason).
  4. Action logged AdminActionLog records action + reason + admin ID.
  5. Student notified If suspended/removed: email sent with reason and appeal option.

Flow 6 — Admin Featuring a Student/Project

  1. Find student via search in admin panel. View their profile.
  2. Toggle "Featured" On StudentProfile or specific Project. Set expiry date (optional).
  3. Student notified "Your profile has been featured on Solvoriz! 🎉"
  4. Recruiter browse Student appears with Featured badge. Sorted higher in results.

Flow 7 — College/Community Onboarding

  1. Admin creates invite Generates unique signup link: solvoriz.com/join/iit-bombay
  2. Shared with college POC at college shares link in WhatsApp group, email blast, notice board.
  3. Students signup via link Institution tag auto-applied to all profiles from this link.
  4. Admin tracks "IIT Bombay — 47 signups, 31 discoverable profiles."
  5. College cohort visible to recruiters Optional: filter by institution (Phase 2 feature).

Section 15

Guardrails

What to avoid, what to delay, and where solo founders typically derail their own MVPs.

Scope Creep Traps

  • Adding a job posting / application system — turns into a job board, kills positioning
  • Building a student-to-student messaging system — legal liability around minors + massive moderation burden
  • Creating a social feed / "following" system — requires content volume that doesn't exist at launch
  • Adding resume upload — directly contradicts the "projects are the signal" core thesis
  • Building company review features (rate your interview) — scope is a completely different product
  • Adding video profiles — high infrastructure cost, low early value
  • Gamification points / leaderboards — students game the system instead of adding real projects

Bad Feature Ideas for Now

  • Skill endorsements (LinkedIn-style) — meaningless without a large network
  • AI-suggested matches in v1 — needs 1000+ profiles to work properly
  • Recruiter-to-recruiter sharing — niche feature, high complexity
  • School/GPA field on student profile — reintroduces resume thinking
  • Community forum / discussion board — content moderation at scale for teenagers

Data Fields to Avoid

  • GPA / grades — not what this platform is about
  • School name as a required field — creates pedigree bias, against mission
  • Salary expectations — too early, legally complex with minors
  • Phone number — unnecessary, privacy risk for students
  • Home address — never needed, privacy violation risk
  • Gender / demographic data — don't collect what you don't need

UX Mistakes to Avoid

  • Making projects optional in onboarding — students will skip it and create empty, undiscoverable profiles
  • Showing recruiter names/emails to students without contact — invites off-platform communication
  • Infinite scroll without filters — recruiters need structure, not browsing fatigue
  • Making the profile completion score punishing — frame it as "unlock benefits", not "you failed"
  • Showing contact credit count prominently during browse — creates anxiety that slows down adoption

Admin Mistakes to Avoid

  • Approving all recruiters automatically without review — one bad actor destroys student trust permanently
  • Featuring every student who asks — featured = editorial signal, must be earned
  • Sending moderation emails that feel punishing — always frame as "here's how to fix this"
  • Collecting metrics you don't act on — pick 5 metrics and watch them daily
  • Over-automating moderation before you understand the patterns — false positives damage good students
